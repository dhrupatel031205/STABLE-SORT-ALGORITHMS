ü´ß Bubble Sort ‚Äî Theory and Implementation

üìñ What is Bubble Sort?

Bubble Sort is a simple comparison-based sorting algorithm that repeatedly steps through the list, compares adjacent elements, and swaps them if they are in the wrong order. The algorithm gets its name because smaller elements "bubble" to the top of the list.

üîß How It Works:

  1. Pass through array from left to right
  2. Compare adjacent elements at positions i and i+1
  3. Swap if necessary: if element at i > element at i+1, swap them
  4. Repeat until no swaps are needed in a complete pass
  5. Each pass moves the largest unsorted element to its correct position

‚ú® Stability in Bubble Sort:

Bubble Sort is stable because it only swaps adjacent elements when the left element is strictly greater than the right element (using > instead of >=). This means equal elements maintain their relative order from the original array.

üìä Complexity Analysis:

  • Time Complexity: O(n¬≤) in worst and average cases, O(n) in best case (already sorted)
  • Space Complexity: O(1) - sorts in-place
  • Stable: ‚úÖ Yes

üìù Insertion Sort ‚Äî Theory and Implementation

üìñ What is Insertion Sort?

Insertion Sort is a simple sorting algorithm that builds the final sorted array one item at a time. It's much like how you would sort playing cards in your hands - picking up one card at a time and inserting it into its correct position in the already sorted cards.

üîß How It Works:

  1. Start from second element (first element is trivially sorted)
  2. Extract current element as "key"
  3. Compare with previous elements and shift them right if they're larger than the key
  4. Insert the key into its correct position
  5. Repeat until all elements are processed

‚ú® Stability in Insertion Sort:

Insertion Sort is stable because it only shifts elements that are strictly greater than the key element. When elements are equal, the algorithm stops shifting and inserts the key after the existing equal elements, preserving their original order.

üìä Complexity Analysis:

  • Time Complexity: O(n¬≤) in worst and average cases, O(n) in best case (already sorted)
  • Space Complexity: O(1) - sorts in-place
  • Stable: ‚úÖ Yes

üîÄ Merge Sort ‚Äî Theory and Implementation

üìñ What is Merge Sort?

Merge Sort is a divide-and-conquer sorting algorithm that divides the array into two halves, recursively sorts each half, and then merges the two sorted halves back together.

üîß How It Works:

  1. Divide array into two equal halves
  2. Recursively sort each half using the same algorithm
  3. Merge the two sorted halves back together
  4. During merge, choose elements from either half based on comparison
  5. Continue until the entire array is sorted

‚ú® Stability in Merge Sort:

Merge Sort is stable when merging, if elements are equal, the algorithm always chooses the element from the left subarray first. This preserves the relative order of equal elements from the original array.

üìä Complexity Analysis:

  • Time Complexity: O(n log n) in all cases
  • Space Complexity: O(n) - requires auxiliary array for merging
  • Stable: ‚úÖ Yes (with proper implementation)
